# 열화상 카메라 모니터링 및 레시피 기반 촬영 시스템 개발 설계서

## 1. 프로젝트 개요

64대의 열화상 카메라를 챔버(Temperature Calibration Chamber) 내부에서 제어하며, 사용자가 설정한 **레시피(Recipe)** 에 따라 챔버의 환경(온도·습도)을 제어하고, 서보 유닛으로 블랙바디(2개)를 이동시켜 순차적으로 영상을 취득하는 시스템입니다. 
A&D PLC가 챔버와 서보 유닛의 물리적 제어를 담당하며, 중앙의 마스터 애플리케이션(Master Application)이 Modbus TCP를 통해 전체 공정을 통제합니다. 카메라 영상 수신 부하는 다수의 에이전트(Agent) PC로 분산 처리하는 **Master-Agent 아키텍처**를 채택했습니다.

*(이 설계서는 기존의 캘리브레이션 시퀀스 방식에서 **레시피 기반 자동 촬영 방식**으로 전환된 최신 요구사항을 반영하고 있습니다.)*

## 3. 소프트웨어 아키텍처 및 기술 스택

### 3.1 기술 스택
- **언어 & 프레임워크**: C# 12, .NET 8.0
- **UI 프레임워크**: WPF (Windows Presentation Foundation) + `CommunityToolkit.Mvvm`
- **통신 프로토콜**:
  - `FluentModbus` / `NModbus4` (PLC 통신)
  - `Grpc.AspNetCore` / `Grpc.Net.Client` (Master-Agent 통신)
  - `System.IO.Ports` (열화상 카메라 가상 시리얼 통신)
- **영상 처리**: `OpenCvSharp4` (영상 수신 및 이미지 처리)
- **로컬 DB (히스토리/레시피 저장)**: `LiteDB` 또는 `SQLite` + EF Core
- **로깅**: `Serilog` (파일 기반 구조화 로깅)

### 3.2 프로젝트 구조 솔루션 (HeatingCameraSystem.sln)
```text

 ├── HeatingCameraSystem.Core          # 공통 모델 (레시피, 챔버, 카메라 DTO), 유틸리티
```

## 4. 핵심 도메인 로직: 레시피 기반 시퀀스 (Recipe Engine)

### 4.1 레시피 구조
- **Global Settings**: 타겟 챔버 온도, 타겟 챔버 습도
- **Steps (N개)**: 카메라-위치(블랙바디 이동 위치), 타겟 블랙바디 온도

### 4.2 실행 시퀀스 흐름
사용자가 `Start` 버튼을 누르면 Recipe Engine이 아래 상태 머신(State Machine)을 실행합니다.

```text
[시작: 레시피 실행]
       │
       ▼
  [챔버_설정] 타겟 온도/습도 전송 (Modbus)
       │
       ▼
  [챔버_대기] 도달 완료 감지까지 대기
       │
       ▼
  ┌▶ [스텝_시작] 카메라 매핑 위치로 서보 이동 명령
  │    │
  │    ▼
  │  [이동_대기] 위치 도달 확인
  │    │
  │    ▼
  │  [블랙바디_온도설정] 타겟 온도 설정
  │    │
  │    ▼
  │  [블랙바디_대기] 타겟 온도 도달 대기
  │    │
  │    ▼
  │  [영상_취득] Agent로 촬영 명령(gRPC) 전송
  │    │
  │    ▼
  │  [이미지_저장] 로컬 원본 및 Master 전송
  │    │
  │    ▼
  └──[다음_스텝_판단] ──(잔여 스텝 존재 시 반복)
       │
       ▼ (모든 스텝 완료)
[종료: 레시피 완료]
```

### 4.3 블랙바디 운영 전략
- **기본 모드**: 1개의 블랙바디를 이동시키며 한 카메라씩 촬영. (나머지 1개는 웜업(Warm-up) 용도로 활용하여 다음 스텝의 온도를 미리 맞춤)
- **동시 촬영 모드(옵션)**: 활성화 시 2대의 블랙바디를 동시에 사용하여 2대씩 촬영 속도 단축.

## 5. PLC 제어 스펙 (A&D 회의록 반영)

Master Application은 PLC와 오로지 Modbus TCP를 통해서만 상호작용합니다.

| 기능 분류 | 동작 상세 설명 | 제어 방식 (Modbus) |
|---|---|---|
| **운전/정지** | 챔버 프로그램 컨트롤러의 ON/OFF를 제어 | Write Coil / Register |
| **온도 제어** | 설정온도(SV) 쓰기 및 현재온도(PV) 읽기 | Read/Write Holding Register |
| **습도 제어** | 챔버 내부 습도를 인지(PV)하여 설정값(SV) 기준으로 유지 | Read/Write Register |
| **N₂ Gas 투입** | 결로 방지용. 온도 하강 시 습도 설정값 초과할 경우 비례 제어로 N2 Gas 투입 | 자동 동작 (PLC 내장 로직) / 모니터링 |
| **상부 배기 밸브** | 내부 습도 상승 시 오픈, 설정 습도 도달 후 클로즈 (과습 시에만 동작). 주위 온도가 높아 온도 상승 구간에서는 동작 안함 | 자동 동작 (PLC 내장 로직) / 모니터링 |
| **서보 유닛 (블랙바디)** | 마스터에서 1~64 카메라 위치값을 지정하면 해당 위치로 이동. 도착 시 완료 신호 회신 | Write Register (위치 명령) / Read Coil (도착 플래그) |
| **비상 멈춤 (E-Stop)**| 발동 시 히터, 냉동기, 부하측 OFF 및 서보 유닛 전원 차단. (서보 전원 차단 후에도 위치 기억 보존) | Write Coil (비상 정지 명령) |

## 6. UI 구현 스펙 (Master Application)

### 6.1 메인 대시보드 (실시간 모니터링)
- **영상 업데이트**: 1초 간격 (Agent에서 수집된 영상을 압축 후 Master로 전송하여 렌더링).
- **View Modes (반응형 렌더링)**:
  - Mode 1: 전체 자동 순환 (페이지당 최대 8개, 1초마다 자동 페이징).
  - Mode 2~5: 사용자가 드래그앤드롭한 특정 카메라 (1, 2, 4, 8개 분할 뷰) 전용.
- **컨트롤 패널**: 타겟 챔버 온도/습도 실시간 차트 제공, 선택된 레시피 Start/Stop 기능.

### 6.2 레시피 편집 및 매핑 화면
- **레시피 편집**: 스텝 단위(카메라 지정, 타겟 온도) 드래그앤드롭 순서 변경 및 CRUD.
- **카메라-위치 매핑**: 64개의 물리적 그리드(Grid) UI에 검색된 Agent 카메라 리스트를 드래그하여 배치.

### 6.3 히스토리 (검색 및 조회)
- **조회 조건**: 날짜/시간 구간, 특정 카메라 번호 (온도 범위 조건은 폐기).
- **표시 내용**: 취득된 온도/습도/이미지 썸네일 리스트. 이미지 클릭 시 모달(Modal) 형태의 팝업으로 원본 확대.

## 7. 데이터 저장 정책 (Data Retention)

1. **저장 위치 분리**: Master PC와 각 Agent PC의 저장 경로는 사용자가 개별 설정 가능.
2. **저장 포맷**: 사용자가 세팅에서 이미지 포맷 선택 (Raw Binary, 16bit TIFF 등).
3. **보존 정책 (Retention Rule)**:
   - **Agent PC**: 사용자가 지정한 보존 기한(예: 30일) 이후 오래된 이미지 파일 자동 삭제 스케줄러 가동.
   - **Master PC**: 용량 문제 방지를 위해 '무한 보존' 또는 '지정일 이후 삭제' 선택 기능 제공.

## 10. 개발 단계 (Phases 및 마일스톤)

전체 개발은 시스템의 안정성과 연동성을 단계적으로 확인하기 위해 총 6개의 Phase로 나누어 진행합니다.

| Phase | 단계명 | 핵심 목표 및 작업 내용 | 기간(추정) |
|---|---|---|---|
| **Phase 1** | **Core 시스템 기반 구축** | - 프로젝트 구조 세팅 (Master, Agent, Core)<br>- 단일 PC 환경에서 카메라 1대 연동 (OpenCvSharp) 및 영상 캡처/저장 | 2주 |
| **Phase 2** | **PLC Modbus 통신 연동** | - FluentModbus 기반 챔버(온/습도) 및 서보 유닛 제어 로직 구현<br>- 가상 Modbus Simulator를 이용한 Read/Write 테스트 | 2주 |
| **Phase 3** | **Master-Agent 분산 통신망** | - gRPC 기반 Master ↔ Agent 통신 아키텍처 구현<br>- Agent 상태 보고 및 Master 영상/제어 명령 전송 테스트 | 3주 |
| **Phase 4** | **Recipe Engine 구현** | - 레시피(스텝) 생성, 저장, 로드 기능<br>- 타겟 온도 도달 대기, 서보 이동, 촬영으로 이어지는 상태 머신(State Machine) 개발 | 2주 |
| **Phase 5** | **WPF UI 및 대시보드** | - 메인 대시보드 (View Mode 1~5 자동 순환 및 분할 뷰)<br>- 레시피 편집기 및 카메라-위치 매핑 드래그앤드롭 UI | 3주 |
| **Phase 6** | **통합 테스트 및 최적화** | - 실제 환경(또는 시뮬레이터 풀 스케일) 64대 연동 및 USB 대역폭 최적화<br>- E-Stop 등 예외 상황 방어 로직 강화 | 2주 |
